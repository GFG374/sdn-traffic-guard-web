# SDN Ubuntu部分文件



## 文件说明

1. **sdn_attack_topology.py** - Mininet拓扑创建脚本
   - 攻击拓扑
   - 在mininet的example文件夹下
   - 启动 --- python3 sdn_attack_topology.py
   

2. **simple_switch_13.py** - RYU控制器应用
   - 基于OpenFlow 1.3的简单交换机控制器
   - 实现基本的二层交换功能
   - 启动 --- ryu-manager --verbose ryu.app.simple_switch_13
   

3. **syn_flood.py** - SDN的syn攻击模块  h3 攻击 h1
   - h3 cp /home/dsw/桌面/syn_flood.py /home/ --- 将攻击文件复制到h3主机里面
   - mininet> xterm h3 --- 进入h3的终端
   - sudo python3 /home/syn_flood.py --- 执行攻击脚本
   - sudo tcpdump -i h1-eth0 tcp port 80 ---开始检测tcp的数据

4. **udp_flood.py** - SDN的udp攻击模块
   - tcpdump -i h1-eth0 -n udp port 53 --- 在h1上开始监听 udp的数据包
   - h4 cp /home/dsw/桌面/udp_flood.py /home/ --- 将攻击文件复制到h4主机里面
   - python udp_flood.py

5. **icmp_flood.py** - icmp的攻击
   - h5 cp /home/dsw/桌面/icmp_flood.py /home/   将攻击文档复制到h5的主机里面
   - python3 icmp_flood.py ---h5的终端执行命令
   - tcpdump -i h1-eth0 -n icmp --- 查看是否有大量的icmp的数据包

6. **botnet_attack.py** - 傀儡机攻击  h7操控h6 攻击h1
   - h6 cp /home/dsw/桌面/botnet_attack.py /home/ 将攻击文件复制到h6主机里面
   - h7 cp /home/dsw/桌面/botnet_attack.py /home/ 将攻击文件复制到h7主机里面
   - python3 botnet_attack.py server h7为服务器
   - python3 botnet_attack.py bot 192.168.1.105 h6为傀儡机器
   - 后续可以操控傀儡机去发送SYN数据包等
   

7. **arp_spoof.py** - arp欺骗 h2欺骗h1 h4的mac地址是h2的mac地址
   - h2 cp /home/dsw/桌面/arp_spoof.py /home/ ---复制arp欺骗的文件到h2的终端里面
   - arp -n --- h1使用arp-你来查看
   
 
   
8. 攻击的处理逻辑：
- 对 TCP_SYN 类型设置连接速率限制防御 SYN Flood
- 对 UDP 类型设置端口流量阈值防御 UDP Flood
- 对 ARP 类型建立 IP-MAC 绑定表防御 ARP 欺骗
- 对 ICMP 类型限制请求频率防御 ICMP Flood
- 对异常 TCP/UDP 流量进行源 IP 信誉评估，识别傀儡机
